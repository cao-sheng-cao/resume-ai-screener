# 简历岗位匹配评分系统 Electron 版 v1.0.2

本版本新增：

1. DeepSeek 模型选择功能。
2. 支持在应用中切换快速模式 / 推理模式 / Pro 模式 / 兼容旧模型名。
3. 支持查看每次评分的 Token 消耗。
4. 排行榜新增“模型”和“Token”列。
5. GitHub Actions 工作流已修复：不使用 npm cache，不发布 Release，只上传 Artifacts。

## 可选模型

- DeepSeek V4 Flash｜快速评分：`deepseek-v4-flash` + thinking disabled
- DeepSeek V4 Flash｜严谨推理：`deepseek-v4-flash` + thinking enabled
- DeepSeek V4 Pro｜高质量评分：`deepseek-v4-pro` + thinking disabled
- DeepSeek V4 Pro｜高质量推理：`deepseek-v4-pro` + thinking enabled
- Legacy deepseek-chat：旧兼容快速模式
- Legacy deepseek-reasoner：旧兼容推理模式

## Token 消耗显示

评分完成后，结果页会显示：

- 输入 Token
- 输出 Token
- 总 Token
- 缓存命中 Token
- 缓存未命中 Token
- 推理 Token
- 本次实际使用模型

Token 数据来自 DeepSeek API 返回的 `usage` 字段。

## GitHub Actions 打包步骤

1. 将项目上传到 GitHub 仓库根目录。
2. 确保根目录能看到：
   - package.json
   - main.js
   - src
   - .github
   - README.md
3. 进入 Actions。
4. 运行 Build Windows Installer。
5. 运行成功后，在 Artifacts 下载安装包。

## v1.0.4 新增：候选人备注、归类、岗位筛选排行榜、Apple 风格 UI

本版本新增：

- 每位候选人可添加备注，例如沟通进展、风险点、下一步动作。
- 每位候选人可设置归类：优先推进、建议推进、待复核、作为储备、不建议推进、已联系、已面试、已淘汰。
- 排行榜支持按岗位筛选，适合“本周只看某个岗位的人才池”。
- 排行榜支持只看本周加入的候选人。
- 排行榜支持关键词搜索姓名、摘要、备注和岗位。
- 导出 CSV 时只导出当前筛选结果。
- 完整 JSON 备份会自动包含候选人备注和归类。
- UI 进一步优化为 Apple 应用风格：玻璃拟态、柔和卡片、圆角、轻阴影、蓝紫渐变主色。

建议用法：

1. 先设置岗位标准并保存。
2. 上传该岗位候选人简历并评分。
3. 排行榜会自动记录候选人对应岗位。
4. 在排行榜里添加备注、调整归类。
5. 使用“按岗位筛选 + 只看本周加入”，即可查看本周这个岗位的人才池。


## v1.0.5 新增：AI 判断严格程度 1-5度

本版本新增“AI 判断严格程度”滑杆：

- 1度｜宽松探索：适合人才池拓展，允许更多相邻经验和潜力匹配。
- 2度｜适度宽松：适合一般初筛，可把相关经验折算为部分满足。
- 3度｜标准推荐：默认推荐，严格按岗位标准和简历原文判断。
- 4度｜严格证据：适合重点候选人复核，更看重 quota、deal size、客户层级、签约结果等硬证据。
- 5度｜极严格硬筛：适合高价值岗位或终面前硬筛，只承认直接、明确、可验证证据。

同一份简历前后评分需要尽量保持同一模型、同一岗位标准、同一严格度，避免分数不可比。


## v1.0.6 新增：候选人姓名识别与大致年龄推断

本版本新增：

- 评分结果自动显示候选人姓名
- 根据本科、硕士、博士年份、毕业年份、第一份全职工作年份或累计工作年限推断大致年龄
- 年龄只显示为粗略估计，例如“约35-39岁”或“待推断”
- 显示年龄推断依据和年龄推断置信度
- 排行榜、CSV 导出、JSON 完整备份均保留年龄推断相关字段

注意：年龄推断仅用于招聘沟通时了解候选人大致资历阶段，不应作为录用或淘汰依据。若简历年份信息不足，系统会显示“待推断”。
